# Codebase Review — 2026-07-11

Full three-dimension review of the v1.3 codebase (level-100 arc, save v9), performed by three
parallel read-only review agents (core code correctness / UI+UX / game mechanics+data), each
verifying findings against source — the core and mechanics reviews additionally ran the real
engine in `node vm` sandboxes. Baseline: commit `2e70d6e`, all ten suites green before review.

**Triage key:** `[v1.3.1]` = fixed in the v1.3.1 bugfix release (branch `v1.3.1-bugfixes`);
`[deferred]` = accepted for a later cycle (v1.4 / mobile-UI pass / backlog); severity as
reported by the reviewing agent.

---

## Part 1 — Core code correctness

### Major

1. **[v1.3.1] Sidebar "Camp" ambush soft-locks the UI.** `index.html:210-215`,
   `js/ui/screens.js:2059` (correct handling exists at `screens.js:1015-1029`). The
   Actions-panel Camp handler never navigates to the battle screen on a camp ambush; the
   battle lock in `navigate()` then blanks every nav action. ~14% of sidebar camps
   (0.35 event × 0.4 ambush share) strand the player on a stale screen; only a page reload
   (silently discarding the battle and its first-strike damage) recovers. Independently found
   by the UX review (its #1). Fix: mirror the Explore-screen ambush handling, plus make
   `navigate()` redirect to the battle screen instead of silently returning.

2. **[v1.3.1] `Game.World.learnTech()` accepts `classOnly` techs → `trainingPoints = NaN`
   (persistent corruption).** `js/core/world.js:411/397/414`. `learnableTechs()` filters
   `classOnly` for the UI but `learnTech()` only rejects `monsterOnly`; the 15 classOnly techs
   have no `trainingCost`, so `x < undefined` passes and the subtraction yields NaN — verified
   in a vm: unlimited free learning afterwards, NaN persists to the save. Reachable via the
   documented console surface. Fix: reject `classOnly` in `learnTech()`.

### Minor

3. **[v1.3.1] Fury tick denied when the winning kill levels the player.**
   `js/core/battle.js:763/783/891-894`. The Fury check reads `c.level` after XP is applied;
   a kill that levels you is judged against the new level. Verified empirically. Fix: compare
   against the pre-XP snapshot already taken for `levelDiff`.

4. **[v1.3.1] Current HP/Energy can exceed max after class deactivation/slot swap.**
   `js/core/classes.js:185-187/155-158`, `js/core/character.js:225-232`. `recalcDerived()`
   adjusts maxes but never clamps current values; verified 90/50 over-max HP persisting to
   the save. Fix: clamp current to max in `recalcDerived()`.

5. **[v1.3.1] `monsterFled` battle end never ticks shrine buffs (nor persists).**
   `js/core/battle.js:214-221` vs 694/772/910/988 — the only battle-end phase that skips
   `tickShrineBuffsOnBattleEnd`, player-favorable. Fix: tick + persist, with comment.

6. **[deferred] Energy-cost gating inconsistent between actions.** `battle.js:373-375/390/656`
   vs 514/718: with 1–4 energy a basic attack/item still works at partial cost while techs and
   Defend demand full cost; the archived rule reads as full-cost for attack too. Design
   decision touching the difficulty contract — defer to a v1.4 balance pass with a sim.

7. **[v1.3.1] `CURSE_DAMAGE_MULT = 0.75` contradicts "halves" comments.** `js/balance.js:274`
   + two battle.js comments say 50%; code and DESIGN.md §4 agree on −25%. Fix: correct the
   three comments (code stays 0.75). Also found by mechanics review (its m3).

8. **[deferred] Future-version save is silently discarded and then overwritten.**
   `js/core/save.js:212-213`, `index.html:295-305`. `migrate()` returns null for v10+, boot
   falls through to the creation wizard, and the first persist overwrites the newer save.
   Low likelihood (single local deployment); needs a load-error sentinel spanning save.js and
   boot — defer to next cycle.

9. **[v1.3.1] Vault gold/item ops lack the battle/facility gates every other town action
   has.** `js/core/world.js:288-359`. Only the town UI contains it today; a direct call
   mid-battle would sidestep the death gold penalty and camp robbery. Fix: add both gates.

10. **[v1.3.1] Save failures are silent to the player.** `js/core/save.js:22-31`,
    `index.html:75-77`. localStorage errors log to console only; every `Game.persist()` call
    ignores the boolean. Fix: surface one alert the first time a persist fails.

### Verified clean (core)

RNG discipline (single `_rng`, no stray `Math.random`); migration chain v1→v9 complete, no
gaps, transient battle state excluded; no implicit globals (vm-verified: exactly `Game` +
`BALANCE`); index.html load order satisfies every traced cross-module call; all 87 monsters /
256 items / learnable techs data-complete; no duplicate ids. Note: `copy.statuses = []` on
the battle monster (battle.js:41) is dead code.

## Part 2 — User experience / UI code

### Major

1. **[v1.3.1] Sidebar Camp soft-lock** — same as Core #1.
2. **[v1.3.1] Battle log shows oldest lines; newest scroll out of view.**
   `js/ui/screens.js:2030-2034`, `css/theme.css:508-511`. Every action rebuilds the log panel
   with scroll reset to the top; fights past ~14 lines force manual scrolling after every
   attack. Fix: auto-scroll the log to the bottom on render.
3. **[v1.3.1 (minimal)] Over-weight loot claim is an unrecoverable dead end.**
   `js/core/battle.js:1000-1031`, `screens.js:2059/1844`. Near carry capacity the player can
   neither discard (nav locked) nor consume (item list disabled after victory); Continue
   forfeits the loot — including monster-drop-only uniques — with no warning. Minimal fix
   shipped: claim-failure message states the weight shortfall, and Continue with unclaimed

4. **[v1.3.1] Selling has no confirmation — irreversible for uniques and quest materials.**
   `screens.js:1179-1187`, `js/core/world.js:270-284`. One misclick destroys a unique or an

   items (matching Discard's existing confirm).

6. **[v1.3.1] Drop-target highlight gets permanently stuck.** `js/ui/dragdrop.js:24-30` —

7. **[v1.3.1] Invalid drops fail silently** (`screens.js:652-655/687`) — wrong-slot drops

8. **[deferred] Keyboard accessibility effectively absent** for non-button controls (tabs,

9. **[deferred] Alert-based feedback interrupts the core loop** — hunt-miss modal (~5% of
   hunts), double alert on quest turn-in. Era-plausible; revisit in a v1.4 UX pass.
10. **[deferred] Backpack doesn't stack/sort/filter; shop Sell panel duplicates the whole

## Part 3 — Game mechanics / balance / data integrity

### Critical

C1. **[v1.3.1] Two of Eidas' Echo's drops can never drop — Heir of the Echo Legendary is
    unobtainable.** `js/data/monsters.js:1226-1235`: `lore_eidas_final_journal` at
    `chance: 1` sits *before* the appended `quest_eidas_echo_seal` (0.7) and unique
    `sword_skyspire_ember_blade` (0.05); first-hit-wins makes both trailing entries
    effective-0. The append-last convention collided with a guaranteed drop. Fix: reorder so
    the seal and sword roll before the guaranteed journal (verify whatever consumes the
    journal still completes).

C2. **[v1.3.1] "The Trials of Ascension" uncompletable in forward play — walls the entire
    tier-2→tier-3 class ladder.** `js/data/quests.js` `trials_of_eldor` (accept L30+)
    requires materials whose only source is L10 `estari_ruin_warden`; the 5-level loot
    cutoff (`battle.js:768-776`) means they drop only at player L8–14. Fix (systemic, also
    fixes C3/M3): quest-material (`quest_`-prefixed) drops bypass the 5-level loot cutoff —
    tagged `[revised]` in DESIGN.md (XP/gold/normal-loot cutoff unchanged).

C3. **[v1.3.1] "Vaultbreaker's Reckoning" has the same impossible window** (accept 33+,
    `quest_leviathan_scale` drop window [25..29], and the one bankable scale competes with
    recipe `synth_vault_bulwark_plate`). Fixed by the same quest-material cutoff exemption.

### Major

M1. **[v1.3.1] Arkan characters locked out of their own start town (and racial questline)
    at L1–13.** `character.js:92` starts Arkan in Saratus; `areas.js:212` gates Saratus at
    minLevel 14 and `world.js:91` has no home-town exception, so the L1/6/8 Arkan quests
    can't be accepted/turned in until L14. Fix: `travelTo` exempts the character's home town.

M2. **[deferred → v1.4] Tech ranks 6–9 of Cleave/Impale (6–8 of Firebolt) strictly dominated
    by rank 5.** OFFENSE_TECH_TAPER (×0.55) applied only to ranks 6+ leaves non-monotonic
    chains (Cleave r5 = 2.9 @ E30 vs r9 = 2.26 @ E46); ranks gate on skill 40–70 + Training
    Points yet should never be bought. Also means the post-launch re-sim modestly understates
    real player power (it modeled tapered r6–9 loadouts). Needs a proper re-taper + re-sim —
    v1.4 balance work, spec against `docs/SPEC-V1.4-GAMEPLAY.md`.

M3. **[v1.3.1] Systemic: collect quests silently go dry via the loot cutoff.** Verified dead
    windows: `professor_flad` completable only at exactly L8; `eldor_dr_ferrier` dead L15+;
    `synthesis_supplies` L16+; `juneros_settlements_1` L27+; `kastengard_investigation` L34+.
    (The six arc-band spine quests are safe — 3 sources each, windows verified.) Fixed by the
    C2 quest-material cutoff exemption.

### Minor

m1. **[deferred] Boss "signature" uniques advertise 5% but effectively drop at ~1.8%** due to
    drop-table shadowing (E[kills] ≈ 55 vs the intended ~20); same shadowing cuts
    `estari_ruin_warden`'s ward fragment 0.6 → 0.149 effective. Authoring-convention issue:
    appended entries pay a compounding shadow tax the nominal numbers hide. Revisit rates in
    v1.4; document the convention hazard.
m2. **[deferred]** `the_oruk` levelMin 5 vs giver town gate 6 — cosmetic mismatch.
m3. **[v1.3.1]** Curse comment drift — same as Core #7.
m4. **[deferred]** Band energy Stones dominate same-band Crystals per gold in shops (Crystals
    are dead stock) — economy tuning, v1.4.
m5. **[deferred]** `polearm_ashwood_spear` damage 7 vs tier formula ~5 — only starter-tier
    outlier, trivial.
m6. **[deferred]** Arc spine quest reward weapons are uniformly levelReq = quest levelMin + 4
    (can't equip the headline reward for 4 levels) — confirm deliberate or retune in v1.4.

### Verified clean (mechanics)

Zero dangling references across ~530 data entities (quests/drops/recipes/areas/classes/
shrines/techs); no duplicate ids; no gold loops or buy-sell arbitrage (SHOP_SELL_RATE 0.5,
quest materials no-trade); regular monsters match hp/damage/energy/xp formulas exactly and
arc bosses match documented premiums; tapered weapons/armor conform precisely; no hunting
gap wider than 3 levels in 1–100; collect-quest grind reasonable (E[kills] ≤ 11) inside
windows; class passives within the ≤+25%/tier discipline (max active damage_pct stack +34%).
Note: Heir of the Echo is the only Legendary with no damage passive — review when C1 lands.

## Overall assessment

The codebase is structurally healthy: pure-state-machine discipline holds, RNG/migration/
load-order conventions are followed, data integrity is excellent, and the economy is sound.
The serious defects are *interaction* bugs between individually-correct systems — the
5-level loot cutoff silently poisoning collect quests (C2/C3/M3), the append-last drop
convention colliding with a guaranteed drop (C1), a UI handler that didn't copy its
sibling's ambush handling (Core #1), and a core API trusting its UI filter (Core #2). The
UX layer's real gaps are where v1.3's content scale outgrew the 1–40-era UI (flat 84-item
lists, unstacked backpack) — deferred to v1.4 alongside the mobile-UI spec.
